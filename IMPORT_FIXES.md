# 1Password Import Fixes

The `sauce import 1password` command has been updated to correctly handle 1Password Login items (`categoryUuid: "001"`) and items that do not have explicitly defined titles.

## Summary of Changes

### 1. `Login` items are now imported as `map` secrets
Previously, `Login` items were hardcoded to behave identically to `Password` items (`005`): the script would isolate the designated `password` field, save it as an `environment` secret, and silently discard the username and any associated custom fields.

Logins (`001`) have now been moved to the `map` processing block. This ensures that:
- Usernames and passwords are included as key-value pairs in the resulting map.
- Any custom section fields (e.g., "First Name", "Last Name" attached to the 1Password item) are preserved.
- The `Password` category (`005`) remains mapped to the `environment` type, acting as a simple, single-string secret.

### 2. Smarter Naming Fallback for untitled items
When a 1Password item lacks a `title` field, `normalizeKey` would return an empty string. The previous logic immediately fell back to naming the secret `item_` followed by the first 8 characters of the item's UUID (e.g., `item_omr4zfop`).

The key generation logic has been updated with a smarter fallback cascade:
1. **Title:** Attempt to use the item's `title`.
2. **URL:** If the title is missing, attempt to use the first URL attached to the item.
3. **Username:** If the URL is missing, attempt to use the value of the field designated as `username`.
4. **UUID:** Only if all of the above are empty will it fall back to the `item_<uuid>` pattern.

### 3. Broadened Category Detection
The `OnePUXItem` JSON struct has been updated to read from both `categoryUuid` and `categoryUUID`. This change ensures that the import command is resilient against minor casing variations in the JSON export format generated by different versions of the 1Password client.

### 4. Documentation Updated
The `README.md` was updated to reflect these changes, explicitly separating the documented behavior of the `login` and `password` categories.
